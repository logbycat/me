『C++OJ练习』 Above Average

Description It is said that 90% of frosh expect to be above average in their class. You are to provide a reality check. The first line of standard input contains an integer C, the number of test cases. C data sets follow. Each data set begins with an integer, N, the number of people in the class (1 <= N <= 1000). N integers follow, separated by spaces or newlines, each giving the final grade (an integer between 0 and 100) of a student in the class. For each case you are to output a line giving the percentage of students whose grade is above average, rounded to 3 decimal places. Input Output Sample Input 5 5 50 50 70 80 100 7 100 95 90 80 70 60 50 3 70 90 80 3 70 90 81 9 100 99 98 97 96 95 94 93 91 Sample Output 40.000% 57.143% 33.333% 66.667% 55.556% [cc lang=”java” tab_size=”2” lines=”40”]

#include
#include
using namespace std;

int main()
{
    int c=0, n=0, sum=0; double per=0,ave=0, count = 0;
    while (cin >> c) {
        for (int i = 1; i <= c; i++) {
            ave = 0; sum = 0; per = 0; count = 0;
            cin >> n;
            int* arr = new int[n];
            for (int j = 0; j <= n-1; j++) {
                cin >> arr[j];
                sum = sum + arr[j];
            }
            ave = (double)sum / n;
            for (int k = 0; k <= n-1; k++) {
                if (arr[k] > ave)count = count + 1;
            }
            per = count / n;
            cout << setiosflags(ios::fixed) << setprecision(3) << 100 * per << "%" << endl;
        }
    }
    return 0;
}

[/cc] 重点在于iomanip头文件包含的方法setiosflags（ios：：fixed或者scientific）表示浮点数表示或者科学记数法，然后ios：：right/left表示左右对齐，setprecision(n)表示浮点数的精确度为n 还有一个，定义数组的时候不能把数组的元素个数设置为一个未知数，和VB不一样，用int* arr = new int[n]即可解决，原因不明正在摸索。。。
