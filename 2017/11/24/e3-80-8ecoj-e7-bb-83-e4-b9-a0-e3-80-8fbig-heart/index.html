写一写心路历程吧 ①下午OJ课上：（WOC这些题肯定又是超简单然后调格式调一个小时一点意思都没有我一定要选道难的哦这个看起来挺难的就决定是你了！） 看完题：。。。。。。。这tm什么玩意 硬着头皮开始写，首先没有考虑递归，第一反应是用for循环的输出，最后当然gg了，荒废一下午。 ②晚上刷题时：Google和Baidu都用了真的搜不到这个题，开始考虑怎么写搜到教程一篇，思路很像，链接：http://blog.csdn.net/u013647382/article/details/52752715 ，归纳如下 这种成阶的次方的题是有递归的套路的，即 一.寻找中心点 二.递归到画自己附近的图形 三.直接写出最低阶的情况 首先得考虑不管怎样都要使用二维数组，因为你要是一行一行输出绝对会崩溃掉。。。 一.寻找中心点 这个题中中心点当然就是心形的中心的部分，推理归纳出坐标：一阶（4，2）二阶（25，7）三阶（172，32）再接着推出一般情况（用等比数列求和公式找到的）： （(pow(7, n) + 1) / 2，(pow(5, n) + 3) / 4） 二.递归画自己想要的图形 假设我们已经写出了一个能画出一阶图形的函数draw了，该怎样推一般呢？我们假设一个阶数n，当n为1的时候就可以调用draw函数来输出一个一阶心形，而当阶数大于1的时候，我们就递归调用自身并使阶数减一。如下代码：

arr[x - 2][y - 1] = arr[x - 1][y - 1] = arr[x + 1][y - 1] = arr[x + 2][y - 1] =
