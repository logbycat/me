leetcode每日一题 转变数组后最接近目标值的数组和 2020.6.14
发表于 2020-06-14| 更新于 2020-06-14|Leetcode
阅读量:

题目描述

给你一个整数数组 arr 和一个目标值 target ，请你返回一个整数 value ，使得将数组中所有大于 value 的值变成 value 后，数组的和最接近 target （最接近表示两者之差的绝对值最小）。

如果有多种使得和最接近 target 的方案，请你返回这些整数中的最小值。

请注意，答案不一定是 arr 中的数字。

解答

这道题用二分查找做，把更换值之后的数组和看做一个函数，很明显这个函数是递增的，我选的值越大，得到的数组和就越大。我想要枚举出一个值，用二分会快一些。因此用二分来解决这个问题。

然后先讲一下二分的细节，因为这个地方实在很恶心，等于开区间+1什么的。

二分

这是二分查找的框架。

  1. int right = nums.length - 1;

    这里初始化的是nums.length - 1，相当于两端闭区间，就是[left, right]，假如用nums.length，理解为小于这个数，也就是左闭右开，[left, right)。核心概念，搜索区间，也就是[left, right]的区间，在后面也用的到。

  2. while(left <= right)

    之前解释了，考虑到搜索区间的关系，即搜索区间为空的时候停止搜索， [left,left]这一区间必须要处理，不处理就是缺失，所以这里必须用小于等于。非要不写等于也可以，但是要打上补丁，最后判断left这个位置的数。

  3. left = mid + 1 和 right = mid - 1

    这个地方又要考虑搜索区间这个问题。我们的搜索区间是[left,right]，我们已经搜索了middle点，所以下一次搜索就不用考虑中间的点，直接搜索 [left, mid - 1] 或者 [mid + 1, right]。可能会有疑问就是假如这个数不在数组里，会不会越过？假如在4.5这个位置对应target，现在的区间是[4,7]，那么4这个地方的区间永远不会+1，也就是说7会变成5，之后变成4。

二分查找变种

我想查找左侧边界，例如[1,2,2,2,2,3]，正常查的话不一定查到最左边那个2，需要一些特殊的操作。

这里就是修改操作了。首先是上面的搜索区间变成左闭右开操作，其次是right = mid，结合左闭右开，就是搜到了也要从左边开始再搜，因此可以搜到最左边。
